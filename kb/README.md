# Example Kubernetes GitOps Runbooks

This directory contains dummy runbooks intended to look like the sort of knowledge-base content a platform team might sync into Klaudia.

The runbooks focus on a Kubernetes platform operated primarily through GitOps workflows such as Argo CD or Flux.

Included examples:

- `gitops-drift-investigation.md`
- `argocd-sync-failure.md`
- `flux-reconciliation-failure.md`
- `ingress-dns-tls-triage.md`
- `crashloop-after-config-change.md`

These are intentionally generic, but written to be realistic enough for demos and testing.
